The pros and cons of this method... on the one hand it let's us style our HTML without having to alter the HTML at all (by adding a class for example). However, the downside is that it's a bit more rigid.

A caveat to our :last-of-type pseudo-class selector. It selects inside every parent element. It selects the last p tag in every container element. To style only specific first paragraph elements, well you could use descendant or child selectors which is what we do.

We can use "ID selectors" instead of class selectors. They work exactly the same except only one element per page can have a given ID. You cannot really reuse styles as a result. As a result, ID selectors aren't really used. Well to amend what we're saying, IDs are sometimes used in URL fragments (which are useful for directing people to a specific part of our web page). But this is actually more reason to avoid using ID selectors since you may inadvertently break the style when changing the URL.

One last thing we should talk about is CSS specificity. We know that all else being equal rules are applied in a predictable manner from top-to-bottom. However everything isn't equal. Below will be listed from greatest precedence to least precedence.

  1. #button-2 (ID selectors)
  2. .button:link (pseudo-class selectors on a class)
  3. a:link and .synopsis em (pseudo-class selectors on an HTML element and descendant selectors are equal)
  4. .button (class selector)
  5. a (regular HTML selector)

This gets very confusing. It's so confusing an entire methodology called BEM has been created to make everything a class selector. This eliminates specificity issues.
