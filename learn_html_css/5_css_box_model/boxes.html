The CSS box model treats every HTML element as a box with a bunch of properties which determine where it appears on the page.

Some important rules about boxes:

  • Block boxes appear below the previous block element.
  • Width of block boxes is set based on the width of its parent container (in this case the width of the browser window).
  • The default height of block boxes depends on the content it contains.
  • Inline boxes don't affect the vertical spacing of the page. They can have a height (through padding), but it will not cause other elements to shift.
  • The width of inline boxes is based on the content it contains, not the width of the parent of the parent element.

Of course we can change these defaults by using the CSS property "display". Which we can do to change our inline elements to block boxes rather than inline boxes. However it is commented out in the CSS.

Differences between padding and margin:

  • Padding has a background while margins are always transparent.
  • Padding is included in the click area of an element. Margins are not.
  • Margins collapse vertically. Padding does not.

As noted above, inline boxes don't really have a height. Therefore changing the margin on them only affects the horizontal spacing. However the padding can have a height.

A quirk of the box model is veritcal margin collapse. When you have two boxes with vertical margins sitting next to each other they will collapse (instead of being added to each other). Only the biggest one is displayed. This behavior comes in handy when defining the minimum space between elements.

However sometimes you want to prevent this behavior. To do that, you need to put an invisible element between them. In this case we are using the div element. This forces the top margin and bottom margin to activate.

Quick reminder that padding never collapses. So if you can use it, maybe use it?

Finally a third option is to stick with bottom-only or top-only margin conventions.

Now you may be asking: what is div? Well div (and span which we'll now introduce) are container elements because sometimes you just need a generic box to style your web page. They don't provide any semantic meaning like HTML normally does. They provide a hook for adding CSS styles to arbitrary sections of a web page e.g. preventing margin collapse or grouping the first few paragraphs together so as to give them different formatting.

Now what's the difference between div and span? div is meant for block-level content and span is meant for inline content.
